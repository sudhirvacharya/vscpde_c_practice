# Interrupt Notes — General & ARM MCU

---

## 1. What is an Interrupt?

An **interrupt** is a signal to the processor indicating that an event needs immediate attention. When an interrupt occurs, the CPU:

1. Suspends the current task (saves context)
2. Jumps to a special routine called an **Interrupt Service Routine (ISR)**
3. Executes the ISR
4. Restores context and resumes the interrupted task

---

## 2. Types of Interrupts

### 2.1 Hardware Interrupts
Generated by external hardware devices (e.g., UART, GPIO, Timer, ADC).

- **Maskable Interrupts (IRQ)** — Can be enabled/disabled by software.
- **Non-Maskable Interrupts (NMI)** — Cannot be disabled; used for critical events (e.g., power failure, watchdog reset).

### 2.2 Software Interrupts
Triggered by software instructions (e.g., `SVC` in ARM, `INT n` in x86).

### 2.3 Exceptions
Internal CPU events:
- Division by zero
- Page fault / Memory access violation
- Undefined instruction
- Stack overflow

---

## 3. Interrupt Lifecycle

```
Normal Execution
      |
  [Interrupt Occurs]
      |
  CPU finishes current instruction
      |
  [Save Context] → Push PC, PSR, registers onto stack
      |
  [Fetch ISR address from Vector Table]
      |
  [Execute ISR]
      |
  [Restore Context] → Pop registers from stack
      |
  Resume Normal Execution
```

---

## 4. Key Concepts

| Term | Description |
|---|---| Forerunner 165?
| **IRQ** | Interrupt Request — hardware signal to CPU |
| **ISR** | Interrupt Service Routine — handler function |
| **Vector Table** | Memory table holding ISR addresses |
| **Interrupt Latency** | Time from interrupt trigger to first ISR instruction |
| **Context Saving** | Saving CPU state (registers, PC, flags) before ISR |
| **Interrupt Priority** | Order in which simultaneous interrupts are handled |
| **Preemption** | Higher-priority interrupt can interrupt a lower-priority ISR |
| **Interrupt Nesting** | Multiple interrupts running inside one another |
| **Critical Section** | Code region where interrupts must be disabled |
| **Polling vs Interrupt** | Polling = CPU checks device status; Interrupt = device notifies CPU |

---

## 5. Interrupt Priority

- Each interrupt source is assigned a **priority level**.
- Lower priority ISRs can be preempted by higher priority ones.
- Priority levels vary by architecture (e.g., ARM Cortex-M has 8–256 levels).

---

## 6. Interrupt Masking

Masking disables specific (or all) interrupts temporarily:

```c
// Generic pattern (ARM Cortex-M)
__disable_irq();   // Disable all interrupts (set PRIMASK)
// Critical section code here
__enable_irq();    // Re-enable interrupts
```

---

## 7. General ISR Best Practices

- **Keep ISRs short and fast** — minimize work inside ISR.
- **No blocking calls** — no `delay()`, no `printf()` inside ISR.
- **Use flags/queues** — set a volatile flag in ISR, handle logic in main loop.
- **Volatile variables** — declare shared variables as `volatile` to prevent compiler optimization.
- **Avoid dynamic memory** — no `malloc()`/`free()` inside ISR.
- **Atomic operations** — use atomic reads/writes for multi-byte variables.

```c
volatile uint8_t data_ready = 0;

void UART_ISR(void) {
    rx_buffer = UART->DATA;   // Read data
    data_ready = 1;           // Set flag
}

int main(void) {
    while (1) {
        if (data_ready) {
            data_ready = 0;
            process_data();   // Handle outside ISR
        }
    }
}
```

## 8. ARM Cortex-M Interrupt System

### 8.1 NVIC — Nested Vectored Interrupt Controller

The **NVIC** is the interrupt controller in ARM Cortex-M cores. It provides:

- Up to **240 external IRQ** sources
- Configurable **priority levels** (3–8 bits depending on implementation)
- **Nested interrupts** — higher-priority ISRs can preempt lower ones
- **Tail-chaining** — back-to-back ISR execution without full context restore
- **Late arrival** — higher-priority interrupt handled first if it arrives during stacking

### 8.2 ARM Cortex-M Exception Types

| Exception # | Name | Type | Priority |
|---|---|---|---|
| 1 | Reset | Fault | -3 (highest) |
| 2 | NMI | Fault | -2 |
| 3 | HardFault | Fault | -1 |
| 4 | MemManage | Fault | Configurable |
| 5 | BusFault | Fault | Configurable |
| 6 | UsageFault | Fault | Configurable |
| 11 | SVCall | System | Configurable |
| 14 | PendSV | System | Configurable |
| 15 | SysTick | System | Configurable |
| 16+ | IRQ0–IRQ239 | External | Configurable |

### 8.3 ARM Vector Table

Located at address `0x00000000` by default (can be relocated via `VTOR` register).

```
Address     | Content
------------|------------------
0x00000000  | Initial Stack Pointer
0x00000004  | Reset Handler address
0x00000008  | NMI Handler address
0x0000000C  | HardFault Handler address
...         | ...
0x00000040  | IRQ0 Handler address
0x00000044  | IRQ1 Handler address
...         | ...
```

### 8.4 Context Saving on ARM Cortex-M

On interrupt, hardware **automatically pushes** to stack:

```
Stack Frame (pushed by hardware):
  xPSR    (Program Status Register)
  PC      (Return address)
  LR      (Link Register)
  R12
  R3
  R2
  R1
  R0
```

> Registers R4–R11 are **not** automatically saved — the ISR must save them if needed.

### 8.5 NVIC Registers (Cortex-M)

| Register | Address | Purpose |
|---|---|---|
| `NVIC_ISER` | `0xE000E100` | Interrupt Set Enable |
| `NVIC_ICER` | `0xE000E180` | Interrupt Clear Enable |
| `NVIC_ISPR` | `0xE000E200` | Interrupt Set Pending |
| `NVIC_ICPR` | `0xE000E280` | Interrupt Clear Pending |
| `NVIC_IPR`  | `0xE000E400` | Interrupt Priority |
| `SCB->VTOR` | `0xE000ED08` | Vector Table Offset Register |

### 8.6 NVIC Configuration (CMSIS)

```c
#include "stm32f4xx.h"   // Or device-specific header

// Enable IRQ
NVIC_EnableIRQ(USART1_IRQn);

// Set Priority (lower number = higher priority)
NVIC_SetPriority(USART1_IRQn, 2);

// Disable IRQ
NVIC_DisableIRQ(USART1_IRQn);

// Set Pending (software trigger)
NVIC_SetPendingIRQ(USART1_IRQn);

// Clear Pending
NVIC_ClearPendingIRQ(USART1_IRQn);
```

## 9. ARM Priority Grouping

ARM Cortex-M supports splitting the priority byte into **preemption priority** and **sub-priority** bits.

```c
// Set priority grouping: 4 bits preemption, 0 bits sub-priority
NVIC_SetPriorityGrouping(4);

// Encode priority: group 4, preemption=2, sub=0
uint32_t priority = NVIC_EncodePriority(4, 2, 0);
NVIC_SetPriority(USART1_IRQn, priority);
```

| Group | Preemption Bits | Sub-Priority Bits |
|---|---|---|
| 0 | 0 | 4 |
| 4 | 4 | 0 |
| 7 | 7 | 0 |

## 10. Special ARM Registers for Interrupts

| Register | Purpose |
|---|---|
| `PRIMASK` | Disables all interrupts except NMI and HardFault (`__disable_irq()`) |
| `FAULTMASK` | Disables all interrupts except NMI |
| `BASEPRI` | Disables interrupts at or below a set priority level |
| `CONTROL` | Selects privileged/unprivileged mode and stack pointer |

```c
// Using BASEPRI to mask low-priority interrupts
__set_BASEPRI(0x50);   // Mask priority >= 5 (depends on priority bits)
// Critical section
__set_BASEPRI(0);      // Restore
```

## 11. SysTick — System Tick Timer

SysTick is a 24-bit countdown timer built into Cortex-M, used for RTOS ticks and delays.

```c
// Configure SysTick for 1ms interrupt (assuming 72 MHz clock)
SysTick_Config(72000);   // 72000 ticks = 1ms

void SysTick_Handler(void) {
    system_tick++;   // Increment tick counter
}
```

## 12. PendSV and SVC

### PendSV
Used by RTOS for **context switching**. Set to lowest priority so it runs after all other ISRs.

```c
// Trigger PendSV from software
SCB->ICSR |= SCB_ICSR_PENDSVSET_Msk;
```

### SVC (Supervisor Call)
Software interrupt for **OS system calls** or switching from unprivileged to privileged mode.

```c
// ARM Assembly SVC call
__asm("SVC #0");
```

## 13. Interrupt-Driven UART Example (STM32)

```c
// Enable UART RX interrupt
USART1->CR1 |= USART_CR1_RXNEIE;
NVIC_EnableIRQ(USART1_IRQn);
NVIC_SetPriority(USART1_IRQn, 1);

// ISR
void USART1_IRQHandler(void) {
    if (USART1->SR & USART_SR_RXNE) {
        rx_data = USART1->DR;   // Read clears RXNE flag
        data_ready = 1;
    }
}
```

## 14. Interrupt Latency Factors

- **Pipeline flush** after branch to ISR
- **Context save** time (hardware stacking)
- **Memory wait states**
- **Interrupt priority** and queuing
- **Tail-chaining optimization** (ARM) reduces re-stacking between back-to-back ISRs

Typical ARM Cortex-M latency: **12–16 cycles** from interrupt assertion to first ISR instruction.

---

## 15. Common Mistakes

- Forgetting to **clear interrupt flag** in ISR → endless loop
- Not declaring shared variables as `volatile` → compiler caches stale value
- Too much work inside ISR → increased latency for other interrupts
- Incorrect **priority configuration** → unexpected preemption behavior
- Re-entrant ISR issues without proper guarding
- Stack overflow due to deeply nested interrupts

---

## 16. Quick Reference — ARM Cortex-M Interrupt Setup Checklist

- [ ] Configure peripheral to generate interrupt (e.g., UART RXNEIE bit)
- [ ] Set ISR priority with `NVIC_SetPriority()`
- [ ] Enable IRQ with `NVIC_EnableIRQ()`
- [ ] Write ISR function with correct name (matches vector table)
- [ ] Clear interrupt flag inside ISR
- [ ] Declare shared variables as `volatile`
- [ ] Test with both normal and edge-case conditions
